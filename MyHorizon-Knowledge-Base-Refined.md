## Communication Philosophy

### "Socratic Questioning" Approach
Before proposing solutions, we ask:
- "What specific outcome are you trying to achieve?"
- "What's the cost of NOT solving this problem?"
- "What manual process takes the most time in your business?"
- "If you could automate one thing tomorrow, what would it be?"

### First Principles Thinking
Break down every client request into:
1. **Core Business Need** - What problem actually needs solving?
2. **Current Manual Process** - How is it done now, and where are the pain points?
3. **Simplest Technical Approach** - What's the minimum viable automation?
4. **Scalability Path** - How does this grow with the business?

### "Agile Yoga" Philosophy
- **Flexible but Structured** - Adapt to client needs while maintaining process rigor
- **Rapid Iteration** - Ship MVP, gather feedback, improve quickly
- **Realistic Optimism** - Excited about possibilities, honest about limitations
- **Fail Fast, Learn Faster** - Test assumptions early, pivot when needed

## Technical Standards

### Tech Stack Preferences
- **Frontend:** React, TypeScript, Tailwind CSS
- **Backend:** Supabase, Node.js, Python for ML
- **AI/ML:** OpenAI GPT-5, Claude, Gemini, custom fine-tuning
- **Integrations:** Zapier, Make.com, custom APIs
- **Deployment:** Vercel, AWS, Google Cloud

### Code Quality Standards
- TypeScript strict mode, no `any` types
- Component-driven architecture
- Comprehensive error handling
- Mobile-responsive by default
- Accessibility (WCAG AA minimum)

### Security Requirements
- API keys stored securely (Supabase Vault, environment variables)
- Row-level security (RLS) for all database tables
- Authentication required for sensitive operations
- Regular security audits
- GDPR/compliance-ready data handling
